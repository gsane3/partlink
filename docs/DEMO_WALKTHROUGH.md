# Partlink MVP Demo Walkthrough

---

## 1. Σκοπός Demo

Αυτό το demo παρουσιάζει τη βασική ροή του Partlink MVP για request-first αγορά ανταλλακτικού.

**Τι αποδεικνύει το demo:**

- Ο αγοραστής βρίσκει ανταλλακτικό στο marketplace και στέλνει αίτημα ή αίτηση τιμής
- Ο πωλητής λαμβάνει το αίτημα, απαντά και στέλνει τιμή
- Ο αγοραστής αποδέχεται την τιμή
- Ο admin παρακολουθεί τη δραστηριότητα της πλατφόρμας
- Το επόμενο βήμα είναι η παραλαβή ή αποστολή

Η ροή αντικατοπτρίζει τον τρόπο που δουλεύουν πραγματικά οι μάντρες και τα συνεργεία.

---

## 2. Πριν ξεκινήσετε

- Εκκινήστε την εφαρμογή τοπικά: `npm run dev`
- Ανοίξτε: **http://localhost:3000**
- Δεν υπάρχει backend ή βάση δεδομένων. Όλα τα δεδομένα είναι mock / local state
- Ορισμένες ενέργειες (πχ. αποδοχή τιμής, αποστολή τιμής) ενημερώνουν την οθόνη μέσω local component state και δεν διατηρούνται μετά από reload
- Αυτό είναι MVP για επικύρωση. Δεν είναι production σύστημα

## 7. Τι Δεν Ισχύει Ακόμα

Αυτό είναι MVP. Μην υποσχεθείτε τα παρακάτω σε pilot ή επενδυτή:

| Λειτουργία | Τρέχουσα κατάσταση |
|-----------|-------------------|
| Real backend | Δεν υπάρχει. Όλα είναι mock / local state. |
| Αυθεντικοποίηση | Δεν υπάρχει. Οι ρόλοι είναι simulated. |
| Πληρωμές | Δεν υπάρχουν. Τα UI options είναι placeholder. |
| Courier API | Δεν υπάρχει. Η επιλογή αποστολής είναι mock. |
| Persistence μετά reload | Δεν υπάρχει. Οι ενέργειες (αποδοχή τιμής, αποστολή τιμής) γίνονται reset με refresh. |
| VIN compatibility database | Δεν υπάρχει ακόμα. Η αποκωδικοποίηση VIN είναι mock. |
| QR scan σε production | Το UI υπάρχει. Η real scan λογική απαιτεί backend. |

## 8. Checklist Pilot Seller — Ερωτήσεις Αξιολόγησης

Στείλτε αυτές τις ερωτήσεις σε πιλοτικό πωλητή μετά από το demo:

1. **Θα ανεβάζατε ανταλλακτικά με αυτόν τον τρόπο;**
   Είναι εύκολο; Τι θα αλλάζατε στα βήματα;

2. **Καταλαβαίνετε τι κάνει η εισαγωγή με VIN;**
   Θα τη χρησιμοποιούσατε για ένα νέο αυτοκίνητο στη μάντρα;

3. **Η ροή με αίτημα αγοραστή — σας φαίνεται κοντά στο πώς δουλεύετε σήμερα;**
   Συνήθως στέλνουν μήνυμα ή καλούν πρώτα;

4. **Τι πληροφορίες χρειάζεστε πριν στείλετε τιμή;**
   Λείπει κάτι από το αίτημα;

5. **Το QR scan για αποστολή — θα ήταν χρήσιμο στην πράξη;**
   Πώς ελέγχετε τώρα τι φεύγει;

6. **Τι θα σας εμπόδιζε να το χρησιμοποιείτε καθημερινά;**
   Τι είναι απαραίτητο για να αξίζει;
